# Governance

Decision Deliberation is currently maintainer-led. The maintainer owns release decisions, repository administration, security response, and changes to the project's core invariants.

Design changes should be discussed in an issue or Discussion before implementation. Accepted changes should be recorded in the relevant contract, ADR, or product plan. Pull requests require passing checks and maintainer review before merge.

As sustained contributors emerge, the project can add reviewer and maintainer roles based on demonstrated judgment, constructive participation, and reliable stewardship rather than contribution count alone.
